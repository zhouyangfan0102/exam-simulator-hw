# 考试模拟器开发交接说明

更新时间：2026-07-15
当前版本：v1.3.4

## 1. 项目概况

考试模拟器是一个面向 Windows 和 macOS 的本地 Web 应用。用户维护 Excel 题库，双击对应系统的启动文件运行 Python 服务后在浏览器中练习。项目无第三方运行依赖，不联网即可使用。

技术结构：

- Python 标准库 HTTP 服务。
- 原生 HTML、CSS、JavaScript 前端。
- `.xlsx` 文件直接通过 ZIP/XML 结构读取和生成。
- `.docx` 与普通 `.xlsx` 题目清单可以整理为标准题库。
- 考试状态保存在服务内存中，题库文件保存在 `data` 目录。

## 2. 当前已完成功能

- 多题库上传、切换、删除和当前题库记忆。
- 按题干、答案、学科或题型进行模糊搜索。
- 多学科组合，随机 30 题或全量出题。
- 随机模式优先按判断、单选、多选各 10 题均衡抽取，不足时由其他题型补齐。
- 每页显示 1 题，答题卡底部提供上一题、下一题和当前题号/总题数，首题与末题会自动禁用无效方向。
- 键盘 `←/↑` 切换上一题，`→/↓` 切换下一题；输入框、下拉框和可编辑区域内不会误触发。
- 单题提交后立即显示正确或错误，支持黄色题目标记和最终交卷统计。
- 状态栏显示总题数、已答、正确、正确率和用时。
- 大题库导航每组显示 50 题，支持状态筛选和直接输入题号跳转。
- 题目导航兼容超过 100 和 10000 道题的编号长度与分页布局。
- 桌面端和移动端响应式布局；临界宽度下主答题区与题目导航会自动收缩，标题、题干、提交答案和交卷按钮按可用宽度缩放。
- 下载标准题库模板，并可整理 Word/Excel 原始题目清单。
- 自动忽略标准表头、常见表头变体和工作表中的重复表头。
- 选择题支持同单元格连续选项、单元格内换行选项以及后续 C 列分行选项。
- 题型为空时，根据答案和选项自动识别判断题、单选题或多选题。

## 3. 当前题库格式

每个 Excel sheet 表示一门学科：

| 列 | 内容 |
| --- | --- |
| A | 题型，可留空自动识别 |
| B | 正确答案 |
| C | 题干与选项 |

支持的选择题写法示例：

```text
单选题 | A | 下列属于偶数的是： A. 2 B. 3 C. 8 D. 9
```

或：

```text
单选题 | A | 下列属于偶数的是：
       |   | A. 2
       |   | B. 3
       |   | C. 8
       |   | D. 9
```

自动识别题型规则：

- 对、错、正确、错误、TRUE、FALSE 等布尔答案识别为判断题。
- 单个 A-H 答案字母识别为单选题。
- 多个答案字母识别为多选题。
- A 列明确填写判断题、单选题或多选题时，以填写值为准。
- 没有足够证据时显示为“未分类”。

## 4. 关键代码位置

- Excel 读取与题库标准化：`server.py` 中的 `load_question_bank`、`looks_like_header`、`normalize_question_layout`。
- 题型自动识别：`server.py` 中的 `infer_question_type`。
- Word/Excel 清单整理：`server.py` 中的 `parse_text_lines`、`parse_table_rows`、`parse_standard_rows`。
- 随机均衡抽题：`server.py` 中的 `balanced_random_sample`。
- 题干和选项拆分：`static/app.js` 中的 `parseOptions`、`prepareQuestion`。
- 单题渲染和翻页：`static/app.js` 中的 `renderCurrentQuestion`、`showQuestion`。
- 键盘切题：`static/app.js` 底部的全局 `keydown` 监听器。
- 大题库导航：`static/app.js` 中的 `renderQuestionNav`。
- 响应式布局：`static/styles.css` 底部媒体查询。

## 5. 本地运行

在项目根目录执行：

```powershell
python server.py
```

默认地址：

```text
http://127.0.0.1:8000
```

如果端口被占用，程序会在后续端口中寻找可用端口。Windows 用户通常双击 `Windows-启动考试模拟器.bat`，结束时双击 `Windows-关闭考试模拟器.bat`；macOS 用户双击对应的 `.command` 文件。首次运行若提示没有执行权限，在终端执行：

```bash
chmod +x macOS-启动考试模拟器.command macOS-关闭考试模拟器.command
```

## 7. 数据与 Git

- Git 仓库中的 `data/题库.xlsx` 是 9 道题的示例题库。
- 用户上传的其他 `data/*.xlsx` 不纳入 Git。
- `data/bank_meta.json` 和 `data/.server-runtime.json` 是本地状态，不纳入 Git。
- 不要把真实业务题库、访问令牌、浏览器配置或运行日志提交到公开仓库。
- 桌面正式项目与 Git 工作目录是两份目录；修改完成后只同步程序文件，不覆盖桌面项目的 `data`。

## 8. 已知限制

- 当前只允许本机通过 `127.0.0.1` 访问，不支持局域网访问。
- 项目不是独立 EXE，目标电脑需要 Python 3.10 以上版本。
- 考试进度只保存在内存中，刷新页面、切换题库或关闭服务后会丢失。
- 上传题库仅支持 `.xlsx`，不支持旧版 `.xls`。
- 原始题目整理支持 `.docx` 和 `.xlsx`，复杂排版仍需人工检查。
- 题型缺失且答案、选项也不足以判断时，只能标记为“未分类”。

## 9. 接手后的推荐流程

1. 先阅读 `AGENTS.md`、本文件和 `使用说明.docx`。
2. 执行语法检查并启动本地服务，确认基线可运行。
3. 修改前检查 `git status`，修改后查看差异。
4. 根据风险补充 Excel 解析或浏览器交互回归测试。
5. 保留用户题库数据，更新说明文档并按现有版本策略打包。
