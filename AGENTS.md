# 考试模拟器开发规则

## 项目目标

这是一个在 Windows 本地运行的题库模拟考试系统。用户通过 Excel 管理题库，程序负责题库上传、切换、搜索、随机或全量出题、即时判题、答题统计和题目导航。

## 当前基线

- 当前对外版本固定为 `v1.3.3`。除非用户明确要求，否则不要修改版本号。
- 后端使用 Python 标准库，不引入第三方运行依赖。
- 前端使用原生 HTML、CSS 和 JavaScript，不引入构建工具或前端框架。
- 服务仅监听 `127.0.0.1`，默认从端口 `8000` 开始寻找可用端口。
- Windows 10/11 和 Python 3.10 以上是目标运行环境。

## 关键文件

- `server.py`：HTTP 服务、Excel/Word 解析、题库管理、考试与判题 API。
- `templates/index.html`：页面结构和静态资源版本参数。
- `static/app.js`：页面状态、题目解析、答题和导航交互。
- `static/styles.css`：桌面端和移动端样式。
- `static/题库模板.xlsx`：用户下载的标准题库模板。
- `data/题库.xlsx`：仓库内的示例题库。
- `启动考试模拟器.bat`、`关闭考试模拟器.bat`：Windows 启停入口。
- `使用说明.docx`：面向最终用户的部署和操作说明。

## 数据保护

- 不要覆盖、删除或重命名用户的 `data/*.xlsx`，除非用户明确要求。
- 仓库只跟踪 `data/题库.xlsx` 示例文件；其他上传题库属于本地数据，不提交 Git。
- `data/bank_meta.json` 和 `data/.server-runtime.json` 是本地运行状态，不提交 Git。
- 同步到桌面正式项目时，只替换代码、模板、样式和说明文档，保留桌面项目的 `data` 目录。
- 测试上传、删除和切换题库时使用隔离的临时项目，不修改正式题库。

## 题库规则

- 每个 Excel sheet 表示一门学科，A/B/C 列分别是题型、答案和题目内容。
- 支持判断题、单选题和多选题。
- 表头及常见表头变体不能计入题目数量。
- 选择题选项既可与题干写在一个 C 列单元格，也可写在后续 C 列单元格。
- 选项标记支持 `A.`、`A、`、`A）`、`（A）`、`[A]` 和 `A 选项内容` 等常见形式。
- 题型为空时，根据答案和选项自动识别；明确填写的合法题型优先，证据不足时保留“未分类”。
- 判断题答案兼容对、错、正确、错误、TRUE、FALSE 等常见写法。
- 多选题答案按选项集合判定，答案字母顺序不影响正确性。

## 实现约束

- 优先沿用现有函数和数据结构，避免无关重构。
- 保持现有 API 路径和响应字段兼容，除非需求明确要求变更。
- 所有动态文本必须转义后再写入 HTML。
- 大题库只渲染当前题目；题目导航继续采用每组 50 题的分页方案。
- 新交互必须兼顾键盘操作、按钮禁用状态、移动端宽度和超长文本。
- 不在页面中增加大段功能说明；操作说明写入 `使用说明.docx`。
- 修改用户可见功能后，同步更新 Word 说明文档。

## 必做验证

修改 Python 后执行：

```powershell
python -m py_compile server.py
```

修改 JavaScript 后执行：

```powershell
node --check static/app.js
```

涉及题库读取时，至少覆盖：标准表头、表头变体、题型缺失、同单元格选项、分行选项、单选、多选和判断题。

涉及页面交互或布局时，启动隔离端口并使用真实浏览器验证桌面端和 390px 左右的移动端：

```powershell
python server.py 8010
```

验证后关闭测试服务并清理测试浏览器配置、截图、临时题库和运行记录。

## 发布与打包

- 发布前停止桌面正式服务，避免把 `data/.server-runtime.json` 放入压缩包。
- 正式压缩包应只包含运行所需文件，不包含 `__pycache__`、`.pyc`、日志、测试文件和浏览器配置。
- 打包后检查压缩包条目、版本标识、Word 标题和 SHA-256。
- 完成后从桌面正式项目重新启动服务，并通过 `/api/subjects` 和首页响应头验证版本与题库数量。

## Git 约定

- 提交前检查 `git status` 和暂存区内容，避免上传本地题库或运行记录。
- 提交信息用简洁中文或英文描述实际变更。
- 不强制推送，不改写远程历史，不提交密钥、令牌或凭据。
